# Commit style

Conventional Commits:
- `feat:` user-visible feature
- `fix:` bug fix
- `refactor:` internal restructuring, no behaviour change
- `chore:` tooling / infra / housekeeping
- `docs:` documentation only
- `test:` tests only

Subject line ≤ 72 chars, imperative mood.

Commits touching auth, payments, or tile signing MUST include a body paragraph
explaining the security implication.
